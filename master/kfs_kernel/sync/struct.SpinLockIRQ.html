SpinLock that disables IRQ.

Description

This type behaves like a spinlock from the Linux crate. For simplicity of use and implementation. The mapping is as follows:

  • lock behaves like a spinlock_irqsave. It returns a guard.
  • Dropping the guard behaves like spinlock_irqrestore

This means that locking a spinlock disables interrupts until all spinlocks have been dropped.

Note that it is allowed to lock/unlock the locks in a different order. It uses a global counter to disable/enable interrupts. View INTERRUPT_DISABLE_COUNTER documentation for more information.
